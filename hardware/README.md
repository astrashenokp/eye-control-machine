# Фізична машинка — 4WD, керування очима через Wi-Fi

Це друга частина проєкту: те, що зараз рухає 3D-модель у [`web/`](../web/)
(`{forward, turn}`, порахований з погляду), тут крутитиме реальні мотори.
Головний ESP32 піднімає власну Wi-Fi точку доступу й приймає команди прямо
з браузера — жодного кабелю між ноутбуком і машинкою.

**Прочитай цей файл повністю перед тим, як щось паяти чи з'єднувати.**
Порядок дій у розділі 7 не випадковий — де-де переставлені кроки можуть
спалити плату.

---

## 2. Важливо прочитати перед збіркою

### 2.1. TP4056 не заряджає твою батарею правильно

TP4056 — це зарядка для **одного** Li-ion елемента (заряджає до 4.2В).
Твій тримач "2×18650, 7.4В" з'єднує обидві банки **послідовно** — щоб
зарядити такий пакет, потрібен модуль на 7.4В/8.4В (2S), інакше зарядка
або не дозарядить банки, або (гірше) переплутає напруги і зіпсує елементи.

**Що робити** (обери одне):
- **Найпростіше (рекомендую для першої збірки):** не вбудовуй зарядку в
  машинку. Виймай 2×18650 з тримача і заряджай окремим зарядним
  пристроєм для 18650 (Liitokala, Opus і подібні, ~150 грн, дуже
  поширені). TP4056 відклади — знадобиться для іншого проєкту з одним
  елементом.
- **Якщо хочеш заряджати не виймаючи:** доклади до замовлення "2S BMS
  7.4V/8.4V Li-ion charger protection board" (~30-50 грн) і заміни ним
  TP4056.

Цей документ далі рахує, що ти йдеш першим (простішим) шляхом.

### 2.2. Тобі потрібен мультиметр

DC-DC понижувач mini-360 має підстроювальний резистор (маленький
потенціометр) — заводська вихідна напруга випадкова, її треба виставити
самому **до** підключення ESP32. Без мультиметра це зробити безпечно
неможливо. Якщо його ще немає — це ще ~100-150 грн, обов'язковий
інструмент для цієї збірки (і для всіх майбутніх).

### 2.3. Мотори розраховані на 3-6В, а батарея дає 7.4В

TB6612FNG подає на мотори повну напругу з батареї (7.4В) імпульсами
(PWM), тож мотори працюватимуть трохи гарячіше й швидше, ніж їм
"призначено". Для короткої їзди це не проблема, але в прошивці нижче
максимальна шпаруватість (duty cycle) обмежена на рівні ~80%, а не 100%
— це просто й безкоштовно береже мотори, без додаткових деталей.

## 3. Загальна схема системи (блоками)

```
                     ┌─────────────────────┐
                     │   2× 18650 (7.4В)    │
                     │   через KCD1-11      │
                     └──────────┬───────────┘
                                │ 7.4В (сира напруга)
                 ┌──────────────┼──────────────────┐
                 │                                  │
                 ▼                                  ▼
      ┌─────────────────────┐          ┌─────────────────────────┐
      │  TB6612FNG  VM       │          │  DC-DC mini-360          │
      │  (живлення моторів)  │          │  вихід виставлений на 5В │
      └──────────┬───────────┘          └───────────┬─────────────┘
                 │                                    │
       AO1/AO2 → 2 мотори (лівий борт)                │ 5В
       BO1/BO2 → 2 мотори (правий борт)                │
                                          ┌─────────────┼──────────────┐
                                          ▼             ▼              ▼
                                   ESP32 DevKit   TB6612FNG VCC   ESP32-CAM
                                   (VIN)          (логіка)        (5V pin)
```

Обидва ESP32 і логіка TB6612FNG живляться від одного 5В виходу
понижувача. Мотори — напряму від батареї (через TB6612FNG VM).

---

## 4. Живлення — точні з'єднання

1. **Батарея → тумблер:** плюс з тримача 2×18650 → один контакт
   KCD1-11 → інший контакт тумблера → далі йде як "PWR+". Мінус з
   тримача — напряму як "PWR–" (тумблер стоїть тільки на плюсовому
   проводі).
2. **PWR+ / PWR–** розводяться на дві гілки:
   - Гілка А (мотори): `PWR+` → `TB6612FNG VM`, `PWR–` → `TB6612FNG GND`
   - Гілка Б (електроніка): `PWR+` → `DC-DC mini-360 IN+`, `PWR–` →
     `DC-DC mini-360 IN–`
3. **Калібрування понижувача (зроби це ДО кроку 4!):**
   - Підключи тільки батарею до входу mini-360, нічого ще не підключай
     до виходу.
   - Постав мультиметр на вихід (OUT+/OUT–).
   - Крути підстроювальний резистор на платі, поки мультиметр не
     покаже рівно **5.0В**.
   - Тільки після цього переходь до кроку 4.
4. **Вихід понижувача (5В) розводиться на трьох споживачів:**
   - `mini-360 OUT+` → `ESP32 DevKit VIN`, `OUT–` → `ESP32 DevKit GND`
   - `mini-360 OUT+` → `TB6612FNG VCC`, `OUT–` → `TB6612FNG GND`
     (той самий GND, що й у батареї — **всі GND мають бути з'єднані
     разом**, це критично для TB6612FNG)
   - `mini-360 OUT+` → `ESP32-CAM 5V`, `OUT–` → `ESP32-CAM GND`

> **Спільний GND.** Батарея, TB6612FNG, обидва ESP32 і понижувач мусять
> мати всі свої GND з'єднані в одну точку (спаяй їх разом на PCB-платі
> в один вузол). Без спільної землі сигнали з ESP32 на TB6612FNG просто
> не працюватимуть.

---

## 5. Мотори + TB6612FNG — сигнальні з'єднання

Мотори лівого борту (2 шт.) підключаються **паралельно** до одного
каналу, правого борту — до іншого:

- Лівий передній мотор + лівий задній мотор → обидва на `AO1`/`AO2`
- Правий передній мотор + правий задній мотор → обидва на `BO1`/`BO2`

Якщо після першого тесту машинка їде назад замість вперед — не
перепаюй нічого, просто поміняй місцями два дроти мотора на виході
(AO1↔AO2 або BO1↔BO2) або зміни напрямок у коді (див. прошивку нижче).

## 6. ESP32 DevKit → TB6612FNG — розводка пінів

| TB6612FNG | ESP32 DevKit GPIO | Призначення |
|---|---|---|
| STBY | GPIO 4 | увімкнення драйвера (завжди HIGH) |
| AIN1 | GPIO 16 | напрямок, лівий борт |
| AIN2 | GPIO 17 | напрямок, лівий борт |
| PWMA | GPIO 18 | швидкість, лівий борт (ШІМ) |
| BIN1 | GPIO 19 | напрямок, правий борт |
| BIN2 | GPIO 21 | напрямок, правий борт |
| PWMB | GPIO 22 | швидкість, правий борт (ШІМ) |
| VCC | 5В (з понижувача) | живлення логіки |
| GND | спільний GND | — |
| VM | PWR+ (сира батарея) | живлення моторів |

Всі обрані GPIO — звичайні, без спецфункцій при старті плати (не
чіпаємо GPIO 0/2/12/15 — вони впливають на режим завантаження ESP32).

## 7. Порядок збірки (не міняй місцями!)

1. Спаяй/зʼєднай тримач батарей → тумблер → так, щоб на виході тумблера
   був чистий PWR+/PWR–. **Батареї поки не вставляй.**
2. Підключи мультиметр і **прокалібруй mini-360** (розділ 4, крок 3) —
   без батарей у тримачі це неможливо, тож встав їх тільки для цього
   кроку, а потім знову вийми.
3. Спаяй усі GND в один спільний вузол (батарея, TB6612FNG, обидва
   ESP32, понижувач).
4. Підключи TB6612FNG (VM, VCC, GND) — поки без ESP32 і без моторів.
5. Підключи 4 мотори до AO1/AO2 і BO1/BO2.
6. Підключи сигнальні піни ESP32 DevKit → TB6612FNG за таблицею з
   розділу 6. ESP32 поки НЕ живи від батареї — проший його спочатку
   від USB (крок 8).
7. Проший головний ESP32 (розділ 8) через USB від ноутбука, поки він ще
   не підключений до батарейної схеми.
8. Відключи USB, встав батареї, увімкни тумблер. Перевір, що ESP32 і
   TB6612FNG отримують 5В (світяться індикатори плат).
9. Піднеси мотор до столу (щоб колеса не зачепили руку) і перевір
   команди через браузер (розділ 10) — усі 4 колеса мають крутитись у
   правильний бік.
10. Тільки після успішного тесту на столі — став колеса на шасі й
    тестуй їзду.
11. ESP32-CAM прошивається окремо, до основної схеми не підключай, поки
    не прочитаєш розділ 9.

## 8. Прошивка головного ESP32

Код лежить у
[`firmware/eye_car_controller/eye_car_controller.ino`](firmware/eye_car_controller/eye_car_controller.ino).

**Встанови перед прошивкою:**
1. Arduino IDE.
2. У Boards Manager встанови пакет **"esp32" (версія 2.0.x)** від
   Espressif — новіші 3.x мають іншу назву функцій ШІМ, код нижче
   розрахований на 2.0.x.
3. Обери плату **"ESP32 Dev Module"**, порт — той, що з'явився після
   підключення USB.
4. Завантаж скетч.

**Що робить прошивка:**
- Піднімає власну Wi-Fi точку доступу з назвою `EyeCar` (пароль
  `eyecar123` — зміни в коді, якщо хочеш інший).
- Приймає HTTP-запити `GET /drive?forward=X&turn=Y` (ті самі числа, що
  вже рахує `logic.js`/`logic.py` — `forward` від -1 до 1, `turn` від -1
  до 1).
- Перетворює `forward`+`turn` на швидкість лівого/правого борту
  (диференціальне керування) і виставляє напрямок + ШІМ на TB6612FNG.
- Якщо команда не приходить довше 500 мс (з'єднання пропало, браузер
  закрили) — **зупиняє мотори** автоматично. Той самий принцип "no
  face → stop", що вже є в софтверній версії.
- `GET /` — проста сторінка статусу (для перевірки, що плата жива).

## 9. ESP32-CAM — окрема плата

ESP32-CAM **не має власного USB** — прошивається тільки через зовнішній
UART-перехідник (той CH340G, що ти купила).

### 9.1. Підключення для прошивки

| CH340G | ESP32-CAM |
|---|---|
| TX | U0R (RX) |
| RX | U0T (TX) |
| GND | GND |
| 5V | 5V |

Перед подачею живлення **з'єднай GPIO0 з GND** дротом-перемичкою — це
переводить плату в режим прошивки. Після завершення прошивки прибери
цей дріт і натисни кнопку RESET на платі (або перепідключи живлення).

> **Типова проблема:** ESP32-CAM під час прошивки часто "просідає" й
> перезавантажується, бо крихітний стабілізатор на дешевому
> USB-перехіднику не тягне пікове споживання камери+Wi-Fi. Якщо
> прошивка постійно обривається з помилками — постав великий
> конденсатор (100-220 мкФ) між 5V і GND прямо на платі ESP32-CAM,
> або живи плату під час прошивки від окремого 5В джерела, а не від
> USB-перехідника.

### 9.2. Що прошивати

У Arduino IDE: **File → Examples → ESP32 → Camera → CameraWebServer**
(стандартний приклад з пакету "esp32", нічого писати самому не треба).

У коді прикладу зміни:
```cpp
#define CAMERA_MODEL_AI_THINKER   // розкоментуй саме цю модель
...
const char* ssid = "EyeCar";        // та сама точка доступу, що й головний ESP32
const char* password = "eyecar123";
```

Після прошивки ESP32-CAM сам підключиться до Wi-Fi головного ESP32 і
підніме відеопотік за адресою, яку покаже в Serial Monitor (щось на
кшталт `http://192.168.4.2/stream`).

### 9.3. Підключення на роботі

Після успішної прошивки — підключи ESP32-CAM до тієї ж 5В гілки
живлення, що й головний ESP32 (розділ 4, крок 4), і закріпи на шасі.
USB-перехідник більше не потрібен.

## 10. Як браузер керує машинкою

Головний ESP32 після ввімкнення створює Wi-Fi мережу `EyeCar`.

1. На ноутбуці підключись до Wi-Fi `EyeCar` (пароль `eyecar123`).
2. Відкрий у браузері `http://192.168.4.1/` — має з'явитись сторінка
   статусу.
3. Веб-застосунок (`web/`) поки що керує тільки 3D-моделлю. Щоб він
   ще й слав команди на машинку, туди треба додати `fetch()` до
   `http://192.168.4.1/drive?forward=...&turn=...` в тому самому місці
   `processFrame`, де зараз рахується `turn` і `speedMult`. Це наступний
   крок — скажи, коли залізо буде готове й перевірене, і я підключу
   цей виклик у `web/js/main.js`.

## 11. Типові проблеми

| Симптом | Причина | Рішення |
|---|---|---|
| ESP32 не прошивається | не той порт/драйвер CP2102 | встанови драйвер CP210x з сайту Silicon Labs |
| Мотори крутяться, але дуже слабо | забув з'єднати всі GND разом | перевір спільну землю (розділ 4) |
| Один борт їде, інший ні | переплутані AIN/BIN піни або обірваний дріт | перевір таблицю розділу 6 мультиметром на прозвін |
| Колесо крутиться в інший бік | мотор підключений "навпаки" | поміняй місцями 2 дроти мотора на виході TB6612FNG |
| ESP32-CAM не прошивається / скидається | живлення від USB-перехідника не тягне | великий конденсатор на 5V/GND або зовнішнє живлення на час прошивки |
| Wi-Fi `EyeCar` не з'являється | прошивка не запустилась | відкрий Serial Monitor (115200 бод), подивись помилки при старті |
